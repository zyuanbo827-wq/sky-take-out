# Health Consultant - 基于 Langchain4j 与 DeepSeek-V3 的智能健康顾问

一个基于大语言模型的智能问诊与辅助挂号预约系统，旨在解决传统医疗咨询中"挂错号、跑断腿"的痛点。

## 项目简介

本项目利用 **DeepSeek-V3** 强大的语义理解能力，结合 **Langchain4j** 框架，构建了一个具备专业医学知识检索（RAG）与医院业务执行能力（Function Calling）的智能助手。系统不仅能提供基于循证医学的咨询建议，还能直接对接医院号源系统，实现从"问诊"到"挂号"的一站式服务。

---

## 技术栈

- **核心框架**: Langchain4j
- **大语言模型**: DeepSeek-V3
- **向量数据库**: Milvus
- **关键技术**: RAG（检索增强生成）、Function Calling、Prompt Engineering、Hybrid Search

---

## 核心功能与技术亮点

### 1. 高精度 RAG 架构搭建

为了解决大模型在医疗领域的"幻觉"问题，并提升回答的专业度，构建了基于 Langchain4j 的检索增强生成链路。

#### 多路召回策略

- **BM25 召回**: 保证关键词匹配的精确度
- **向量召回**: 捕捉语义层面的相关性
- **RRF（倒数融合） + Rerank**: 对多路结果进行重排序，进一步提升排序质量

#### 技术优势

- **元数据增强**: 在数据预处理阶段注入元数据（如科室、适用人群、发布时间），提升检索的可解释性与精度
- **效果提升**: 相比仅使用向量召回，该策略使 **命中率** 和 **MRR（平均倒数排名）** 提升了 **20%+**
- **幻觉抑制**: 有效抑制了模型的事实性幻觉

### 2. 严谨的 Prompt 工程优化

针对医疗场景对准确性的高要求，设计了严格的系统提示词与约束规则：

#### 核心规则

1. **零容忍幻觉**: 强制模型仅依据检索到的上下文回答问题，严禁利用训练数据中的外部知识进行发散
2. **防御性回复**: 设定兜底规则，当检索内容不足以回答问题时，明确回复"暂无相关医学文献/相关指南"，禁止编造数据、科室或医生信息
3. **可溯源性**: 要求模型在回答中标注依据来源（具体文献或指南名称），增强患者对 AI 建议的信任度

### 3. Function Calling 智能体与流式交互

通过 Function Calling 赋予模型"手脚"，使其具备执行具体业务的能力。

#### 核心功能

- **智能挂号**: 模型能够识别用户意图（如"我想挂明天的呼吸科"），自动调用医院号源查询接口，并辅助用户完成预约
- **流式输出改造**: 优化了后端到前端的响应机制，采用流式输出，显著降低了用户等待时间，提升了对话的流畅度与体验

## 环境准备与运行

### 1. 配置依赖

```
# 安装Maven依赖
mvn clean install
```

### 2. 配置环境变量

```
# 配置DeepSeek API Key
export DEEPSEEK_API_KEY="your_api_key_here"

# 配置向量数据库连接
export VECTOR_DB_URL="milvus://localhost:19530"
```

### 3. 数据准备

```
# 导入医学知识库数据
python data_preprocess.py
```

### 4. 启动项目

```
mvn spring-boot:run
```

## 开发心得

本项目通过 **RAG + Function Calling** 的双轮驱动模式，成功将大模型从"聊天机器人"转化为"业务执行者"。其中，混合检索策略的引入是提升医疗问答质量的关键，而严格的 Prompt 约束则是保障医疗安全的重要防线。

通过本次开发，深刻认识到：

- 在医疗领域，模型的准确性比创造性更重要
- 检索增强生成是解决专业领域知识更新的有效方案
- 流式输出能显著提升用户体验
