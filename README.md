# SkyGo – 基于 Java 的高并发外卖点单与后台管理系统

一个模拟真实高并发交易场景的分布式外卖系统核心模块。项目涵盖商品管理、订单处理及秒杀活动，重点解决了高流量场景下的库存一致性、缓存穿透及服务稳定性问题。

## 项目简介

**SkyGo** 是一个基于 **Spring Boot** 构建的 B2C 外卖交易平台。系统不仅实现了常规的外卖点单流程，更专注于在“秒杀”等高并发场景下的架构优化。通过引入 **Redis** 缓存集群、**Kafka** 消息队列以及 **Lua** 脚本，成功解决了传统单体应用在流量洪峰下的性能瓶颈，确保了数据的一致性与系统的高可用性。

---

## 技术栈

- **核心框架**: Spring Boot, MyBatis
- **数据库**: MySQL
- **缓存与中间件**: Redis, Redisson, Kafka
- **脚本与工具**: Lua Script, Bloom Filter, AOP, ThreadLocal

---

## 核心功能与技术亮点

### 1. 分布式会话与无状态认证

为了解决传统 Session 在 Tomcat 集群环境下的共享难题，并提升系统的水平扩展能力，设计了基于 Redis 的无状态认证方案。

- **Redis 共享会话**: 使用 Redis 存储用户登录状态（验证码与 Token），彻底解决了 Session 无法在服务器集群间同步的问题。
- **双重拦截机制**: 设计了两层拦截器架构：
    - **第一层**: 校验 Token 有效性，拦截非法请求。
    - **第二层**: 配合 `ThreadLocal` 存储当前用户信息，并在请求结束时自动刷新 Token 的有效期，实现“活跃用户自动续期”，提升用户体验。

### 2. 高性能缓存架构

针对外卖系统中“热点数据”（如热门店铺、菜品）查询频繁的特点，构建了多级缓存体系。

- **多样化数据结构**: 根据数据类型灵活选择 Redis 数据结构（String, Hash, Set 等），优化存储效率。
- **Spring Cache 整合**: 结合 Spring Cache 注解简化缓存开发流程，实现声明式缓存管理。
- **性能提升**: 经压测，热点数据的查询响应时间降低了 **90%+**，显著减轻了 MySQL 数据库的压力。

### 3. 缓存一致性与稳定性保障

在引入缓存后，重点解决了数据一致性（数据库与缓存不一致）及缓存异常（穿透、击穿）问题。

- **一致性策略**:
    - **主动更新**: 对于实时性要求极高的数据，利用 **Kafka** 消息队列进行异步解耦，确保数据库更新后缓存能及时失效或更新。
    - **被动过期**: 对于一般数据，采用 **Redisson** 读写锁策略，防止缓存击穿导致数据库瞬间压力过大。
- **防穿透设计**: 引入 **布隆过滤器 (Bloom Filter)**，在缓存层之前拦截非法 Key 查询，有效防止恶意请求穿透至数据库层。

### 4. 高并发秒杀与库存管理

针对“秒杀”场景下的超卖与并发竞争问题，设计了多层防护机制。

- **乐观锁解决超卖**: 在数据库层面利用版本号机制（乐观锁）更新库存，确保在高并发写入时库存数据的绝对准确。
- **悲观锁一人一单**: 针对用户维度的并发限制，使用悲观锁逻辑确保同一用户在秒杀活动中只能下一单，防止刷单行为。
- **异步削峰**: 引入 **Kafka** 将秒杀请求进行异步化处理。用户请求进入消息队列后立即返回，后台消费者服务按照数据库处理能力平滑消费请求，极大地提升了系统的吞吐量。

### 5. AOP 面向切面编程实践

为了提高代码的可维护性与复用性，大量使用了 AOP 技术。

- **公共字段自动填充**: 通过自定义注解与切面，自动处理 `create_time`, `update_time`, `create_user` 等公共字段的填充，减少了 Controller 和 Service 层的样板代码。
- **统一缓存处理**: 封装通用的缓存操作逻辑，通过拦截器统一处理缓存的读取与写入，降低了业务逻辑与缓存代码的耦合度。
